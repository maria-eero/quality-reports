How It Works

The Pattern: Golden Fixture + Scenario Matrix

Each platform has a single JSON fixture file containing every documented API scenario. Tests load the fixture, parse it with the production decoder, and assert the model matches expected values.

// iOS — one test from the suite:
@Test("CT-007: Security on, non-eero detected — BLOCKED, data disabled")
func ct007_blockedNonEero() throws {
    let connections = try loadConnections()          // loads PortSecurity.json
    let port = try interface(7, in: connections)     // scenario on interface 7
    let security = try #require(port.portSecurity)

    #expect(security.capable == true)
    #expect(security.enabled == true)
    #expect(security.status == .blocked)
    #expect(port.disabledProtocols.contains(.data))
}

// Android — same scenario:
@Test
fun `CT-007 security on, non-eero detected parses as BLOCKED with data disabled`() {
    val port = firstPortOf("scenario_07_blocked_non_eero")

    assertEquals(true, port.portSecurity?.capable)
    assertEquals(true, port.portSecurity?.enabled)
    assertEquals(PortSecurityStatus.BLOCKED, port.portSecurity?.status)
    assertTrue(port.disabledProtocols.orEmpty().contains(PortProtocol.DATA))
}

Why This Instead of Pact

This Approach (Golden Fixture)Pact (Consumer-Driven Contracts)
Backend team involvementNone — fully client-sideRequired — provider must run verification
New infrastructureNone — runs in existing test suitePact Broker + provider CI job
New dependenciesZeroPact-Swift / Pact-JVM + broker client
Time to first valueDays (write fixture + assertions)Weeks (broker setup, provider onboarding)
What it catchesDecoder breaks on unexpected API shapesSame + bilateral deploy-time gating
What it doesn't catchSilent API drift (until fixture is refreshed)Nothing (full contract enforcement)
MaintenanceUpdate fixture when API intentionally changesNegotiate contract changes with broker
Graduation pathFixtures → Pact consumer expectations (low effort)Already at full maturity
Trade-off summary: Pact is stronger (bilateral enforcement, deploy gating) but requires backend participation we don't have today. The golden-fixture approach gives us 80% of the value with 0% of the dependency — and the fixtures become Pact consumer expectations if we graduate later.

What This Catches That Doesn't Exist Today

Toggle State Mismatch

If the port-security toggle emits before device state resolves, INT-002/003 fail — the ViewModel's emitted state must reflect the polled backend state, not stale cached values.

Directly prevents CORE-31743 pattern (July snapshot)

New Enum Value Crash

CT-M4 asserts that "FUTURE_STATE" decodes without crashing. If someone changes the enum to a failable init, this test catches it.

Directly prevents CORE-31436 pattern

Schema Drift

If a dev accidentally changes the model in a way that breaks parsing of the documented scenarios, the CI test fails before merge.

Guards every documented scenario in the tech spec matrix
